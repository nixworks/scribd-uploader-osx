Information about the Scribd Uploader source code

The source code for Scribd Uploader is fully documented using Doxygen. You can download and install Doxygen and use it to generate documentation for the Scribd Uploader code to assist you while developing. Doxygen generates documentation in many formats, but the configuration file (“Doxyfile”) provided with the source code enables HTML and DocSet output only.

The HTML output can be viewed in any web browser, and the DocSet output is automatically copied to Xcode’s documentation folder when generated, making it available right in Xcode. A token file is also generated so that Research Assistant can be used to provide pop-up documentation when developing Scribd Uploader.

You can install the Mac OS X Doxygen application, install it via MacPorts, or compile and install Doxygen from source.

If you already have Doxygen installed, you can skip this next section to learn how to generate the Scribd Uploader documentation.

To use the Doxygen Mac OS X front-end:

  1. Visit http://www.stack.nl/~dimitri/doxygen/download.html#latestsrc in your web browser.

  2. Download and install the application available from the disk image under “A universal binary distribution for Mac OS X 10.5 (Leopard).”

  3. Open the Scribd Uploader.xcodeproj project in Xcode.

  4. Edit the Run Script phase of the Documentation target.

  5. Change the line /opt/local/bin/doxygen to the path to your doxygen tool.

    This tool is located within the Doxygen.app application you just installed. If, for example, you installed it into the Applications folder, the tool’s path would be /Applications/Doxygen.app/Contents/Resources/doxygen.

To install Doxygen from MacPorts:

  1. Install MacPorts if you have not already done so.

    Installation instructions can be found by visiting http://www.macports.org/install.php in your web browser.

  2. Open Terminal and run the command sudo port install graphviz +no_x11 +no_pangocairo doxygen.

To compile Doxygen from source:

  1. Visit http://www.stack.nl/~dimitri/doxygen/download.html#latestsrc in your web browser.

  2. Download and install the archive under “The source distribution.”

  3. Visit http://www.stack.nl/~dimitri/doxygen/install.html#install_src_unix in your web browser and follow the instructions to complete the installation.

  4. Open the Scribd Uploader.xcodeproj project in Xcode.

  5. Edit the Run Script phase of the Documentation target.

  6. Change the line /opt/local/bin/doxygen to the path to your doxygen tool.

    If you used the default installation settings, the tool’s path would be /usr/local/bin/doxygen.

With Doxygen installed, you can now generate the HTML and DocSet documentation for Scribd Uploader.

To generate and view the source documentation for Scribd Uploader:

  1. In Xcode, activate the Documentation target.

  2. Choose Build under the Build menu.

    Xcode will run the shell script that generates the documentation.

  3. Once the build is complete, quit and re-launch Xcode.

    You only have to do this the very first time you build the Documentation target.

  4. Choose Developer Documentation under the Help menu.

  5. Under the Home toolbar menu, you should now see a “Scribd Uploader” help section. Select it.

    You must re-run the Documentation target if you change the documentation for any header file and want to see those changes reflected in the DocSet.
