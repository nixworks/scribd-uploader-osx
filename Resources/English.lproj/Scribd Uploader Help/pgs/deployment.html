Deploying new versions of Scribd Uploader

Preparing a copy of Scribd Uploader for release is a two-step process, described below.

To build the release files for Scribd Uploader:

  1. Build using the Install configuration in Xcode.

  2. At the command line, run the Distribution/MakeDMG.rb script.

    This script will create a new disk image containing the release application, adorn it with a custom icon and background, compress it, add a license agreement, and prepare it for Internet distribution.

The release application and disk image will be in build/Install/dist. The disk image is ready to be distributed online.

Automatic updates

Scribd uses Sparkle for automatic updating. If you distribute your own version of Scribd Uploader, by default it will use Scribd.com to check for updates, and therefore only see official updates (instead of your updates).

If you want your users to see your updates, you will have to syndicate your own appcast, and update the Info.plist file to read from it, instead of the Scribd.com appcast. See the Sparkle documentation for information on how to do this.
