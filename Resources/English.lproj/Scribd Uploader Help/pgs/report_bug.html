Reporting a bug or crash

Scribd Uploader is never intentionally released with significant bugs, although occasionally we may elect to release the product with minor bugs. View a list of bugs we already know about.

If you find a bug or other unexpected behavior in Scribd Uploader, you can report it to us by emailing the lead developer, Tim Morgan, at tim@scribd.com. Please provide as much information as you feel might be helpful. If you are familiar with Mac OS X applications like Console, information from those utilities can also be helpful.

If Scribd Uploader crashes, Finder will display a dialog allowing you to report the crash. Unfortunately, this reports the crash to Apple, not to us, so it doesn’t do us very much good. You can, however, still get us the information, by following the steps below, once the crash notification dialog appears.

Reporting a Scribd Uploader crash to Scribd:

  1. Click the Report… button.

    A Problem Report dialog will appear.

  2. Click the “Problem Details” tab.

  3. Select all the text in the large text field, then choose Edit > Copy to place it in your clipboard.

  4. Open your email client and create a new email. Address it to tim@scribd.com.

  5. Choose Edit > Paste to place the crash log into your email.

    You may also want to include other information, such as what you were doing to make it crash, or other unusual details about the situation.

  6. Send the email.

If you are a Mac OS X developer familiar with Cocoa and Objective-C, you might even be able to fix the bug yourself! Read Add to or modify the code for Scribd Uploader for more.
