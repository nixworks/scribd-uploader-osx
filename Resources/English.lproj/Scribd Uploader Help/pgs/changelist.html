List of changes

Each new version of Scribd Uploader introduces changes: new features or bug fixes that differentiate it from the previous version. For reference, a comprehensive list of changes is included in this help file. As each new version is released, its changes are added to this file. Changes are listed in reverse chronological order, with the current (most recent) version of the program appearing first.

Version 3.0 (June 31, 2010)

Scribd Uploader now requires Mac OS X 10.6 (“Snow Leopard”). Version 3.0 of the Uploader takes advantage of the features introduced in this version of Mac OS X.

New Features

  • Redesigned user interface.

  • A Publisher metadata panel has been added where information about the publisher and license of the document can be specified.

  • You can now upload files on the Internet by URL. See Adding a file by URL for more information.

  • Scribd Uploader will now check if it’s being run in the Applications folder, and prompt the user if it is not.

  • When no network connection is available, Scribd Uploader now prompts you to use the Network Diagnostics tool.

Feature Improvements

  • The Information drawer now fades from a pre-upload to a post-upload state, or vice versa.

Bug Fixes

  • Fixed a crash that would occur when logging in with no network connection.

  • Fixed a crash that would occur when trying to view files on the website or view upload errors.

  • Fixed a bug that would cause all files to have the same status badge, even when they had different statuses.

  • Fixed a bug that would cause the Information drawer not to update if a file was uploaded while being selected.

  • Fixed a bug that would sometimes cause file names to be displayed incorrectly.

Version 2.1.1r2 (February 12, 2009)
Bug Fixes

  • Fixed a bug where dragging folders into the file list would cause the sheet to hang.

Version 2.1.1 (February 12, 2009)
New Features

  • A Dock menu has been added allowing users to add files to upload from the Dock.

Feature Improvements

  • The Information drawer now displays view and edit buttons that take you to the Scribd.com website if you select a document that has already been uploaded.

  • Dragging a folder into the file list will cause Scribd Uploader to scan that folder for documents.

  • The dock icon will now bounce when uploads are complete if Scribd Uploader is not the active application.

Bug Fixes

  • The sheet that appears if a file is moved or deleted between launches is now correctly placed over the file list window.

  • If Scribd Uploader is launched on a machine running a version of Mac OS X prior to 10.5, a warning message is displayed and the program exits.

  • If a file is placed where the application support folder should be, the user is presented with a friendlier error message.

  • If this version of Scribd Uploader is used with a newer data format, a more friendly error message will be displayed.

Version 2.1 (January 30, 2009)
New Features

  • Information drawer added; users are able to tag and categorize their documents before uploading, as well as set other metadata. See Editing information about your files for more information.

    The Upload as private checkbox has been removed. To upload a document as private, use the new checkbox in the Information drawer. See Uploading files that only I can see for instructions.

    The sheet that appears when uploads are complete has been removed, as it is no longer necessary to edit your documents on Scribd.com.

  • A feature has been added that allows users to add all the uploadable documents in a folder and all its subfolders.

  • The success checkmark has been replaced with an arrow that takes the user to the Scribd document when clicked.

    In addition, a button will now appear once all uploads are complete. This button will take the user to their Scribd.com My Docs page, where they can view their documents on the Web.

  • A “Upload as private by default” preference has been added to the Preferences window. Users who primarily upload private documents can now have that setting be enabled for new documents by default.

  • Rather than store Scribd.com session keys or passwords in plain text, Scribd Uploader now creates a secure keychain item.

    If you have used a previous version of Scribd Uploader and you are concerned about others accessing your plain-text password, remove the com.scribd.Uploader.plist file in the Home > Library > Preferences folder.

Bug Fixes

  • Users with special characters in their passwords can now log in.

  • A bug preventing Quick Look from working was fixed.

  • Users with Growl installed will now see Growl notifications when their uploads complete.

  • Optimizations to reduce the amount of memory Scribd Uploader uses.
